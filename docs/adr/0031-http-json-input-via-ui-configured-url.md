# ADR-0031: HTTP(JSON)入力を、UIで設定するURL+固定名スカラーとして追加する

- Status: accepted
- Date: 2026-07-07

## Context

ブラウザ上で完結する処理系である以上(ADR-0006)、`fetch()` で任意のHTTP
エンドポイントからデータを取ってきて作品を動かす、という要望は自然で、
実際に検討した。天気・仮想通貨価格・センサー値のような外部の数値ソースを
uniform として使いたい、というのは audio/OSC と同種の要求。

設計で分かれた点は3つ:

1. **データの種類**: JSON数値のみに絞った。画像URL入力(`webcam` と同様の
   sample テクスチャ)は需要が出たら別ADRで検討する
2. **URLの指定方法**: この言語には `"""..."""`(FFI用の生文字列)しか
   文字列リテラルが無い。ソースにURLを書ける形(生文字列の流用、または
   新しい1行文字列リテラルの追加)も検討したが、**URLはエディタ横のUI
   テキストボックスで設定し、言語側は `http.value` という固定名の
   スカラーだけを見る**ことにした。理由: URLは「作品のロジック」ではなく
   「どのデータソースに繋ぐか」という実行環境の設定に近く、`mouse`/`audio`
   が対応するハードウェアの選択をUI側(ブラウザの権限ダイアログ)に
   任せているのと同じ扱いが自然。言語仕様(レキサ/パーサ)を変えずに済む
   利点も大きい
3. **取得頻度とCORS**: HTTPは1フレーム(16ms)には到底収まらないので、
   OSC/audioと同じ「バックグラウンドで定期取得し、最新値をその都度参照する」
   非同期入力パターンにする。ポーリング間隔は固定5秒(`HTTP_POLL_MS`)。
   CORSは相手のAPIが許可していない限り `fetch()` が失敗するが、
   ブリッジ([[ADR-0029]])経由での回避は今回は見送った(直接
   `fetch()` で十分な需要が大半という判断。CORS非対応APIは使えない
   と割り切る)

## Decision

- `src/runtime/inputs.ts` に `InputEngine.setHttpSource(url, path)` を追加。
  `url` が空なら停止するだけ(前回値は保持、ADR-0010 の精神)。空でなければ
  即座に1回 `fetch(url).then(r=>r.json())` し、以降 `HTTP_POLL_MS`(5秒)
  ごとに繰り返す。`path` はドット区切り(`current.temp_c` 等)で JSON を
  辿る小さなヘルパー `getJsonPath` で解決し、数値でなければ前回値を保持して
  ステータスにその旨を出す
- 取得した値は `values.set("http.value", ...)` として**毎フレーム無条件で**
  書く(`mouse.x`/`entropy` と同じ扱い。`InputAdapter` の登録やコンパイル済み
  プログラムの参照有無によるゲーティングは行わない — URLをUIに入力すること
  自体が既にオプトインなので、`ensure()` 経由の追加ゲートは不要と判断)
- 言語側(`src/compiler/stdlib/inputs.ts`)は `http.value`(`Float`)を
  1つ追加するだけ
- `index.html` にエディタ横の2つのテキストボックス(Data URL / JSON path)
  と、取得結果を表示するステータス欄を追加。`main.ts` は `change` イベントで
  `InputEngine.setHttpSource` を呼び直す。`InputEngine` は device.lost
  ([[ADR-0028]])で作り直されるため、再初期化のたびに UI の現在値を
  渡し直す

## Consequences

- ✅ 天気/価格/センサー等の外部数値APIを `http.value` 1つで作品に取り込める
- ✅ 言語仕様(レキサ/パーサ/型系)を一切変更していない
- ✅ device.lost からの再初期化後もUIの入力値からHTTP入力を復元できる
- ⚠️ 1URL・1パスのみ(複数のデータソースを同時に使うことはできない)。
  需要が出たら複数スロット化(OSCの固定バンクと同様の形)を検討する
- ⚠️ CORSを許可していないAPIは使えない。回避にはブリッジ経由([[ADR-0029]]
  と同じ形)が必要になるが、今回は範囲外
- ⚠️ JSON以外(プレーンテキスト、CSV等)には対応しない
