# ADR-0001: 作品を時空間上の純粋関数とする

- Status: accepted
- Date: 2026-07-04

## Context

creative coding 言語の実行モデルには、フレームループ命令型(Processing)、宣言的関数合成型
(Hydra/TidalCycles)、シェーダ型(GLSL)などの選択肢がある。本言語の目標は
「2D/3D + GPU、ライブ前提、新パラダイム探求」であり、ライブ書き換え時の状態移行問題と
GPU コンパイルの両方を解く実行モデルが必要。

## Decision

作品を `(空間座標, 時間) → 色` の**純粋関数**とし、言語をその関数を合成する代数として設計する。
系譜としては Conal Elliott の Fran(アニメーション = 時間の純粋関数)と
Pan(画像 = 座標の純粋関数)の合流点。

## Consequences

- ✅ ホットスワップが自明に安全(状態がないので関数を差し替えるだけ。ADR-0004)
- ✅ 純粋・エイリアシングなしの関数はそのまま WGSL に落ちる(ADR-0007)
- ✅ 新旧プログラムの同時評価が可能 → クロスフェードが実装できる
- ⚠️ 履歴が必要な表現(トレイル、シミュレーション)は書けない → 状態を限定的に再導入する(ADR-0003)
- ⚠️ 乱数も決定的(`hash`)である必要がある
