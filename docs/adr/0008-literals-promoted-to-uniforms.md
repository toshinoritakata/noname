# ADR-0008: 数値リテラルは uniform に昇格する

- Status: accepted
- Date: 2026-07-04

## Context

ライブコーディングの最頻編集は「数値をいじる」こと。数値変更のたびにシェーダを
再コンパイルすると数十〜数百 ms のレイテンシが乗り、ライブの体感を損なう。

## Decision

ソース上の数値リテラルは定数畳み込みせず、**原則 uniform に昇格**する。
`circle 0.3` → `circle 0.35` の編集では IR の形が変わらないため、
シェーダ再利用+uniform 更新だけで反映される(目標 < 16ms = 1フレーム)。

例外: ループ回数・グリッド数など**構造に効く数(構造定数)は畳み込む**。
構造定数か値定数かは IR 構築時に用途で自動判別する。

## Consequences

- ✅ 数値編集が再コンパイルゼロで即反映 — ライブ体感への投資対効果が最大の一手
- ✅ エディタで数値をドラッグする「スクラブ」UI が無料で手に入る
- ⚠️ uniform バッファが肥大し得る(リテラル数に比例)。定数畳み込みによる
  シェーダ最適化の機会を一部失う — ライブでは正しいトレードオフと判断
- ⚠️ 「同じ形の IR かどうか」の判定(ADR-0007 の hash-consing)が正しさの前提
