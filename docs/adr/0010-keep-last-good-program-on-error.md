# ADR-0010: エラー時は直前の正常なプログラムを維持し、映像を止めない

- Status: accepted
- Date: 2026-07-04

## Context

creative coding、特にライブパフォーマンスでは「エラーで映像が止まる」ことが最悪の失敗。
TidalCycles は評価失敗時に直前のパターンを維持する方式を実証している。

## Decision

- パース/型エラー時は**直前の正常なプログラムが走り続け**、エディタに診断だけ出す。
  これを言語仕様として明記する(実装詳細ではなく仕様)
- パーサはエラーでも AST を返す(エラーノード挿入)——複数診断と部分的な解析のため
- シェーダコンパイルエラーも同様: 生成 WGSL→AST span のソースマップで元コードの
  位置に翻訳して表示し、映像は旧プログラムを維持(ADR-0004 の2スロットがそのまま効く)

## Consequences

- ✅ 「評価が怖くない」——ライブで試行錯誤する心理的安全性の土台
- ✅ 2スロット構造(ADR-0004)と機構を共有できる(新スロットの準備が整わなければ切り替えないだけ)
- ⚠️ エラーに気づかず古いプログラムを見続けるリスク → 診断表示は控えめだが明確に
