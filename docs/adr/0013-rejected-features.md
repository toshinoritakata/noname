# ADR-0013: 採らないと決めたもの(組合せ展開・パターン代数・bbox ほか)

- Status: accepted
- Date: 2026-07-04

## Context

先行研究(Curv / Punctual)と設計検討の過程で、機能として魅力はあるが採らないと
判断したものがある。「なぜ無いのか」を後から再発明しないために記録する。

## Decision

以下を採らない:

1. **マルチチャネルの組合せ展開**(Punctual: `[1,2]+[10,20]` → 4チャネル)
   — 少ないコードで爆発的バリエーションが出るが初見殺し。
   バリエーション生成は明示的な `grid`/`scatter`/`map` に限定する
2. **0/1 マスクの図形表現**(Punctual 方式)— SDF を捨てるとモーフとブレンドが死ぬ。
   Shape は常に距離場(ADR-0002)
3. **Tidal 級のパターン代数**(パターンをデータとして持つ設計)— 複雑度が跳ねる。
   `cycle`/`every`/`1beat` は「`time * cps` を折り畳む式」への展開で足りる。需要を見て再訪
4. **bbox**(Curv は Shape に保持)— レンダ最適化・エクスポート用途であり、
   無限に広がる場が多いライブ用途では必須でない。当面持たない
5. **モナド・型クラス・遅延評価** — 汎用言語が IO や無限構造を扱うための装備。
   `out` しか出口がなく全域性が保証された本言語には過剰
6. **汎用 JS ライブラリ FFI** — 際限がなくなる。入力は InputAdapter(ADR-0012)、
   シェーダは wgsl/glsl ブロック(ADR-0011)に限定
7. **デスクトップネイティブ実装** — Curv の教訓(ADR-0006)

## Consequences

- ✅ 言語とコンパイラが小さく保たれ、M0〜M6 のスコープが現実的になる
- ⚠️ いずれも「需要が実証されたら個別 ADR で再訪」が前提。本 ADR は拒否ではなく延期の記録
