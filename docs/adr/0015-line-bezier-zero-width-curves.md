# ADR-0015: line / bezier は距離ゼロの曲線とし、太さは outline に委ねる

- Status: accepted
- Date: 2026-07-05

## Context

「大量の直線・曲線をレンダリングしたい」という要望を受け、線分・2次ベジエ曲線を
形状プリミティブとして追加することにした。次元は `move`/`<+>` と同じく2D/3D
両対応、用途は (a) `scatter`/`grid` による静的な大量生成(string art 的な模様)、
(b) `simulate`/閉形式の位置場を繋いだ軌跡(トレイル)の両方を想定する。

太さをどう持たせるかで設計の分かれ道があった: `line a b w`(幅を引数に取る)か、
`line a b`(幅0の曲線)+ 既存の `outline w` combinator(implementation.md の
形状合成、`dist = abs(d) - w`)の組み合わせかの二択。

## Decision

**`line a b` / `bezier a b c` は太さの引数を取らず、パス上でちょうど距離0になる
「距離ゼロの曲線」を返す。** 太さは既存の `outline w` にそのまま委ねる
(`line a b |> outline 0.02` で幅 0.04 の線分になる)。

- **次元多相**: 引数の点が vec2 なら 2D、vec3 なら 3D(`move`/`<+>` と同じ判別方法)
- **line** は標準的な線分距離(最近傍点までの距離)
- **bezier(2D)** は2次ベジエの厳密距離(iq の解析解、三次方程式を陽に解く)。
  制御点が縮退(ほぼ一直線)しているときは `dot(b,b)` に 1e-9 の epsilon ガードを
  入れて NaN を防ぐ(「壊れても絵になる」方針、乱数200曲線での数値検証で
  境界誤差 < 1e-9 を確認済み)
- **bezier(3D)** は3制御点が張る平面に投影し、平面内厳密距離(2D の式を再利用)と
  平面外距離をピタゴラス合成した**厳密な**3D距離にする(平面曲線への距離として正しい)。
  3点がほぼ縮退して法線が定まらない場合は `line a c` にフォールバックする

## Consequences

- ✅ 新しい概念(太さ引数)を増やさず、既存の `outline` を再利用するだけで済む
  ——「合成子はすべて関数合成の別名」という設計原則(dream-code.md 総括)に忠実
- ✅ `scatter N \i -> bezier a b c |> outline w |> fill c` で大量の曲線が
  ネイティブに(FFI 無しで)書ける。ADR-0014(scatter の点状パーティクル)と
  同じ `loopShape`/CSG 経路にそのまま乗る
- ✅ CPU 側 IR インタプリタ(interp.ts)に同一アルゴリズムを実装し、SDF 性質
  テスト(境界 ≈ 0・Lipschitz ≤ 1)で数値検証済み(2D/3D 双方、乱数サンプル)
- ⚠️ line/bezier の dist は常に非負(曲線までの符号なし距離)。`cut`/`inter` の
  ような「内部を持つ」前提の合成子と組み合わせると意味が薄い(交差判定には使えるが
  「中身をくり抜く」用途には向かない)。太さを与えて `outline` した後であれば
  通常の Shape と同様に振る舞う
- ⚠️ 大量描画時の性能: 現状は ADR-0014 の instanced sprite 化の対象外
  (point|>move|>fill/glow の連鎖のみを検出する)。line/bezier の scatter は
  今のところ通常の CSG ループを通る。実測で 200〜400 本程度は 60fps を維持できたが、
  数千本規模で重くなったら ADR-0014 と同じ手法(ループ不変式の巻き上げ、または
  instanced 線分描画)を line/bezier にも広げる余地がある(需要が見えたら再訪)
