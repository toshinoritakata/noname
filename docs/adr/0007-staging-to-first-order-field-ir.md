# ADR-0007: Staging(部分評価)で一階の Field IR に落としてから WGSL を生成する

- Status: accepted
- Date: 2026-07-04

## Context

高階関数・let 多相を持つ純粋関数型言語を GPU で走らせる必要がある。Curv は
「部分評価→インライン展開→静的サブセット(SubCurv)→シェーダ」の経路を実証済み。

## Decision

型付き Core を **Staging** で一階・純粋な式 DAG(**Field IR**)に落とす:

1. 全関数適用を β 簡約(インライン展開)
2. `grid`/`scatter`/`map` は静的な要素数でループ展開
3. 次元多相(2D/3D)を単相化
4. IR ノードは hash-consing(構造的ハッシュで共有)

これを支える言語仕様上の制限: **クロージャはコンパイル時展開可能なものに限る、
要素数・グリッド数は静的に決まる、一般再帰なし(全域言語)**。

## Consequences

- ✅ Staging が必ず停止し、出力はそのまま WGSL に落ちる
- ✅ hash-consing 1つの機構から CSE・パス単位の差分コンパイル(3.4)・IR スナップショット
  テストの3つが出る
- ✅ 分類としては「純粋関数型のシンタックスを持つ同期データフロー言語」——
  モナド・型クラス・遅延評価は意図的に輸入しない
- ⚠️ 実行時に関数値が飛び回るプログラムは書けない(汎用関数型言語ではない)
- ⚠️ 大きな `grid` の展開でシェーダが肥大するリスク → 展開コストは M2 で計測
