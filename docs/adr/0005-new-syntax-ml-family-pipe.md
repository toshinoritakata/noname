# ADR-0005: 独自シンタックスの新言語とし、ML 系スタイル+ `|>` パイプを採る

- Status: accepted
- Date: 2026-07-04

## Context

実装アプローチには埋め込み DSL(最速)と独自言語(自由度最大)がある。本プロジェクトの
動機は「新しい表現パラダイムの探求」であり、ライブ編集に最適化した構文まで設計対象。

## Decision

- 独自シンタックスの新言語とする。骨格は ML/Haskell 族
  (`name = expr`、`\i -> ...`、`let ... in`、すべてが式)
- チェーンの主演算子は **`|>` パイプ**(F# 起源)。理由:
  1. ライブ編集の最頻手「チェーン末尾に1語足す」が行末追記で済む(括弧の包み直し不要)
  2. `x |> f = f x` で「合成子はすべて関数合成の別名」という意味論と一致
  3. F#/Elm/Elixir で実績があり初見で読める
- 対抗馬の棄却: `.` チェーンはレコードのフィールドアクセスと衝突、
  `>>` は Punctual の出力ルーティングと意味が衝突(点なし合成用に予約)、
  `$` は流れが右から左でライブ編集に不向き
- Punctual の構文摩擦への対策: 単項マイナスは字句レベルで解決、
  行頭 `|>` の省略(インデント継続)を糖衣として用意

## Consequences

- ✅ 多段 `warp` の重ね掛けなどライブの定番イディオムが気持ちよく書ける
- ⚠️ パーサから作る工数を引き受ける(処理系は薄く保つ。ADR-0006, 0007)
- ⚠️ `|>` は2文字+シフトで打鍵コストが高い → 糖衣の効果は M0 で実測して判断
